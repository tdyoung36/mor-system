🔥 Grass fire prevention — Clean Flow
▸ Never park on dry grass or shrubs — exhaust reaches 1,000°F+
▸ Dispose of cigarettes properly — never throw on ground
▸ Maintain 30-foot safety zone — rake leaves, cut dead limbs
▸ Use approved spark arrester on all combustion equipment
▸ Keep shovel, water bucket, or fire extinguisher on site
🔵 Large-diameter sewer — key factors
▸ Production measured in tons per day (not feet)
▸ Flow cannot be stopped without bypass pumping
▸ Higher flow pumps needed — more water refills required
▸ Step cleaning: jet 20-ft section, check debris, set pull distance
▸ Debris table: 24" pipe ½ full = 8.73 tons per 100 feet

SOP 1 — Work order intake Owner / PM
1. Receive WO from SWBNO → log in system immediately
2. Assign contract, verify scope vs. contract schedule of values
3. Enter all intake fields: location, asset ID, pipe size, priority, PM, supervisor, due date
4. Confirm crew/truck availability before scheduling
5. Set status to "Dispatch Needed" → system alerts dispatcher
SOP 2 — Dispatch & mobilization Dispatcher
1. Complete pre-dispatch checklist — all items confirmed
2. Brief field supervisor on scope, conditions, SWBNO contacts, access
3. Confirm arrival on site → text PM → update status to "In Progress"
4. Emergency WOs dispatch within 2 hours of receipt
SOP 3 — Field operations Field Supervisor
1. JSA completed and signed by all crew before any work
2. Document production daily: LF per segment, crew hours
3. Photos: before, during, after work — text to PM same day
4. Collect disposal tickets — stamped by facility
5. Field report signed by foreman same day
SOP 4 — Google Spaces communication All roles
1. All field updates posted to designated Spaces channel
2. Dispatch assignments posted on dispatch — crews acknowledge
3. Safety events posted to Safety channel immediately
4. Billing ready notices sent when all backup docs confirmed
5. Work complete notices sent with production totals
SOP 5 — Billing & invoicing PM / Billing
1. Verify all 9 billing packet items confirmed before building invoice
2. Correct SWBNO pay item codes, quantities from field logs
3. PM reviews before submission
4. Log submission date, set invoice to "Submitted"
5. Follow up: Day 15, Day 30, Day 45 if unpaid
SOP 6 — Safety incident response All personnel
1. Stop work immediately — ensure all personnel safe
2. Notify Field Supervisor → PM within 15 min → Owner immediately
3. Document in safety log within 2 hours
4. Post to Spaces safety channel immediately
5. Corrective action determined before resuming
6. OSHA report if recordable incident
SOP 7 — Change orders PM / Owner
No work on out-of-scope items without written authorization.
1. Identify scope change → notify PM immediately
2. Document in CO log with description and dollar impact
3. Obtain SWBNO rep signature or email
4. PM countersigns before work proceeds
5. Send CO notice to Spaces → invoice on separate line item with CO #
SOP 8 — Fleet & equipment Supervisor / Owner
▸ Daily pre-trip inspection — all trucks before dispatch
▸ Monthly maintenance log entry required
▸ Defects reported to owner same day → Spaces fleet channel
▸ No unit deployed with unresolved safety defects
▸ CDL / medical cards verified monthly
▸ Tanker endorsement required for vacuum trucks

Step-by-step setup guide
1. Create a Google Space for each channel
In Google Chat: click + New Space → name it (e.g. "MOR Operations", "MOR Dispatch", "MOR Safety") → add your team members.
2. Get the webhook URL
Open the Space → click the Space name at the top → "Manage webhooks" → click "Add webhook" → name it "MOR System" → click Save → copy the URL.
3. Paste and save
Paste each webhook URL into the matching field on the left. Click "Save all webhooks". URLs are stored securely in your browser's localStorage.
4. Test each channel
Click the "Test" button next to each URL to send a test message. Green = working. If you see "CORS blocked" — this is normal for direct browser posting. The system will copy the message to your clipboard automatically so you can paste it in Spaces.
About CORS
Google Chat webhooks are designed for server-to-server use. Most browsers will block direct requests due to CORS policy. When this happens, MOR System automatically copies the formatted message to your clipboard — just paste it in the Google Space. For direct posting without CORS issues, host this file on a server or use a proxy like Pipedream or Zapier.

How to set up Google Spaces webhooks
Step 1 — Create or open a Space
In Google Chat, create a Space for each channel (Operations, Dispatch, Safety, Billing, Fleet, Management).
Step 2 — Add webhook
Inside the Space → click the Space name at the top → "Manage webhooks" → click "+ Add webhook" → name it "MOR System" → copy the webhook URL.
Step 3 — Paste URL above
Paste each Space's webhook URL into the matching field on the left. Click Save.
Step 4 — Send from dropdown
Use the "Send to Spaces ▾" dropdown in the top bar, or the 📤 buttons on any panel, to post updates directly to the right channel.
